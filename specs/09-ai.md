# AI

## 1. Purpose

Analyze enriched events using AI (via Laravel AI SDK) to determine real vs false positive, assess risk, prioritize, and recommend actions. Supports multimodal evaluation (text + media), streaming responses, and a full hybrid pipeline that can operate with or without media evidence.

## 2. Responsibilities

- Evaluate normalized events through a multi-stage pipeline (rules → heuristics → AI textual → multimodal → decision fusion)
- Classify events as real, false positive, noise, duplicate, or unclear
- Score confidence and risk for each evaluation
- Detect false positives using heuristic and AI-based signals
- Recommend prioritized actions (escalate, ignore, create incident, etc.)
- Generate human-readable explanations for every evaluation
- Assess media evidence (video clips, images, audio) when available
- Reevaluate events when new evidence arrives or context changes
- Stream AI agent responses via Soketi WebSocket and SSE fallback
- Record AI inference metrics (tokens, latency, cost) for billing and observability
- Bridge Laravel AI SDK conversations to tenants via conversation links

## 3. Inputs / Outputs

### Inputs

| Source | Data | Channel |
|--------|------|---------|
| Normalization / Context | `NormalizedEvent` + `EventContextSnapshot` | Domain event / direct call |
| Context | `EventMediaContext` (media references) | Eloquent relationship |
| TenantConfig | `TenantAIProfile` (automation_level, token limits) | Service call |
| AI SDK | Agent streaming events (`AgentStreamed`, `AgentPrompted`) | SDK event listeners |
| Scheduler / Listener | Reevaluation triggers (media arrived, context updated) | `AIReevaluationRequested` event |

### Outputs

| Target | Data | Channel |
|--------|------|---------|
| Decisions | `AIEventEvaluation` (classification, risk, confidence, recommended actions) | Domain event `AIEvaluationCompleted` |
| Frontend | Streaming tokens during evaluation | `AIEvaluationProgressBroadcast` on `private-jobs.{taskId}` |
| Frontend | Completed evaluation summary | `AIEvaluationCompletedBroadcast` on `private-accounts.{teamId}` |
| Tenancy | Usage events (`ai_calls`, `ai_tokens_in`, `ai_tokens_out`) | `RecordUsageEvent` action |
| Audit | Inference logs with cost and token breakdown | `ai_inference_logs` table |

## 4. Entities

### 4.1 AI Event Evaluations (`ai_event_evaluations`)

Core evaluation record linking a normalized event to its AI classification, risk, and recommended actions.

```php
Schema::create('ai_event_evaluations', function (Blueprint $table) {
    $table->id();
    $table->foreignId('normalized_event_id')->constrained('normalized_events')->cascadeOnDelete();
    $table->foreignId('team_id')->constrained()->cascadeOnDelete();
    $table->unsignedTinyInteger('evaluation_version')->default(1);
    $table->string('evaluation_mode'); // rules_only, ai_text, multimodal, hybrid, deferred_pending_media
    $table->string('classification'); // real_event, false_positive, noise, duplicate, unclear, pending_evidence
    $table->decimal('confidence_score', 3, 2)->nullable();
    $table->decimal('risk_score', 3, 2)->nullable();
    $table->string('priority_level'); // low, normal, high, urgent
    $table->boolean('is_real_event')->nullable();
    $table->boolean('requires_action')->default(false);
    $table->string('recommended_action')->nullable();
    $table->text('explanation_text')->nullable();
    $table->jsonb('signals_json')->nullable();
    $table->jsonb('evidence_summary_json')->nullable();
    $table->string('model_used')->nullable();
    $table->timestamp('evaluated_at');
    $table->timestamps();

    $table->index(['team_id', 'evaluated_at']);
    $table->index('normalized_event_id');
});
```

**Enum `EvaluationMode`**: `RulesOnly`, `AiText`, `Multimodal`, `Hybrid`, `DeferredPendingMedia`

**Enum `EventClassification`**: `RealEvent`, `FalsePositive`, `Noise`, `Duplicate`, `Unclear`, `PendingEvidence`

**Enum `EvaluationPriority`**: `Low`, `Normal`, `High`, `Urgent`

## 5. Services / Actions

### 5.1 `EvaluateEventWithAI`

**Path**: `app/Domains/AI/Actions/EvaluateEventWithAI.php`

```php
public function execute(NormalizedEvent $event, EventContextSnapshot $context): AIEventEvaluation
```

Full hybrid pipeline orchestrator:
1. Build input context via `BuildAIInputContext`
2. Run quick rules filter (deterministic, no AI call)
3. Run heuristic scoring (risk, recurrence, geofence)
4. If rules-only is sufficient, return early with `rules_only` mode
5. Dispatch AI textual evaluation via `EventEvaluationAgent`
6. If media is available, dispatch multimodal evaluation
7. Fuse all signals into final `AIEventEvaluation`
8. Generate explanation and recommended actions
9. Dispatch `AIEvaluationCompleted` domain event

### 5.2 `EvaluateEventMultimodally`

**Path**: `app/Domains/AI/Actions/EvaluateEventMultimodally.php`

```php
public function execute(AIEventEvaluation $eval, Collection $mediaContexts): AIMediaAssessment
```

- Iterates over `$mediaContexts` (images, video clips, audio)
- Sends each to the AI SDK agent with multimodal prompt
- Creates `ai_media_assessments` records for each
- Returns the aggregate assessment result

### 5.3 `DetectFalsePositive`

**Path**: `app/Domains/AI/Actions/DetectFalsePositive.php`

```php
public function execute(AIEventEvaluation $eval): bool
```

- Checks heuristic signals (recurrence patterns, known false-positive signatures)
- Checks AI classification and confidence threshold
- Returns `true` if the evaluation should be marked as `false_positive`
- Dispatches `FalsePositiveDetected` domain event when detected

### 5.4 `CalculateRiskScore`

**Path**: `app/Domains/AI/Actions/CalculateRiskScore.php`

```php
public function execute(NormalizedEvent $event, EventContextSnapshot $context): float
```

- Combines event severity, asset criticality, driver history, geofence context, time-of-day factors
- Returns a normalized `0.00–1.00` risk score

### 5.5 `GenerateRecommendedActions`

**Path**: `app/Domains/AI/Actions/GenerateRecommendedActions.php`

```php
public function execute(AIEventEvaluation $eval): Collection
```

- Maps classification + risk + priority into a prioritized list of `AIRecommendedAction` records
- Higher risk → more urgent actions (trigger emergency protocol, escalate)
- Lower risk → less invasive actions (log only, wait for media)

### 5.6 `BuildAIInputContext`

**Path**: `app/Domains/AI/Actions/BuildAIInputContext.php`

```php
public function execute(NormalizedEvent $event, EventContextSnapshot $context): array
```

- Structures normalized event data, context snapshot, media references, historical signals, and tenant profile into the input array expected by the AI SDK agent
- Redacts sensitive fields before sending to AI provider

### 5.7 `ReevaluateEventWithNewEvidence`

**Path**: `app/Domains/AI/Actions/ReevaluateEventWithNewEvidence.php`

```php
public function execute(NormalizedEvent $event, string $triggerType): AIEventEvaluation
```

- Creates a new `AIEventEvaluation` with incremented `evaluation_version`
- Runs the full pipeline with updated context/media
- Links to existing reevaluation request record
- Does NOT mutate previous evaluations

## 6. Jobs

### 6.1 `EvaluateEventJob`

- **Queue**: `ai-evaluation`
- **Retry**: 2
- **Timeout**: 120 seconds
- **Payload**: `normalized_event_id`
- **Logic**:
  1. Load `NormalizedEvent` with context snapshot
  2. Check tenant AI quota via `TenantAIProfile` — skip if over limit
  3. Call `EvaluateEventWithAI::execute()`
  4. Record `ai_calls` usage event via `RecordUsageEvent`

### 6.2 `EvaluateEventMediaJob`

- **Queue**: `ai-evaluation`
- **Retry**: 2
- **Timeout**: 180 seconds
- **Payload**: `evaluation_id`, `media_context_ids`
- **Logic**:
  1. Load `AIEventEvaluation` and associated media contexts
  2. Call `EvaluateEventMultimodally::execute()`
  3. Update evaluation classification if media contradicts or confirms

### 6.3 `ReevaluateEventJob`

- **Queue**: `ai-evaluation`
- **Retry**: 1
- **Payload**: `normalized_event_id`, `trigger_type`
- **Logic**:
  1. Load reevaluation request and mark as `processing`
  2. Call `ReevaluateEventWithNewEvidence::execute()`
  3. Mark request as `completed`

## 10. Business Rules

1. Every relevant normalized event MUST pass through the AI evaluation or heuristic pipeline — no event should reach Decisions without an evaluation record.
2. AI evaluation can start without media; if media arrives later, a reevaluation is triggered automatically via `AIReevaluationRequested`.
3. Absence of media does NOT block the pipeline — the evaluation proceeds with `deferred_pending_media` mode and updates when media is available.
4. All evaluations are versioned — reevaluations create new `AIEventEvaluation` records with incremented `evaluation_version`, preserving full history.
5. Explanations are mandatory — every evaluation stores reasoning via the `ai_explanations` table. An evaluation without an explanation is invalid.
6. AI cost is controlled per tenant via `TenantAIProfile` (automation_level, token limits). Evaluations are skipped or downgraded to `rules_only` when quota is exhausted.
7. Fallback to `rules_only` mode if the AI provider is unavailable (timeout, error, rate limit). The pipeline must never fail silently.

## 12. Usage Metering

| Meter Code | Unit | When Recorded |
|------------|------|---------------|
| `ai_calls` | call | 1 per `EvaluateEventJob` dispatch (before AI call) |
| `ai_tokens_in` | tokens | From AI SDK `AgentStreamed` event — input token count |

## 13. Technical Considerations

### AI SDK Agent Configuration

```php
// app/Infrastructure/AI/Agents/EventEvaluationAgent.php
namespace App\Infrastructure\AI\Agents;

use Laravel\AI\Agent;

class EventEvaluationAgent extends Agent
{
    // Defines structured prompt, tools (risk calculator, geofence checker),
    // model configuration, and output schema for structured evaluation results.
    // Supports streaming mode for real-time progress broadcasting.
}
```

### Streaming Architecture

- **Primary path**: `EvaluateEventJob` runs the AI SDK agent in streaming mode. Each streamed chunk is broadcast to `private-jobs.{taskId}` via Soketi.
- **SSE fallback**: `GET /api/{current_team}/ai/tasks/{taskId}/stream` returns a `StreamableAgentResponse` as Server-Sent Events for clients without WebSocket support.
- Cache the `taskId → userId` mapping in Valkey with a 10-minute TTL for channel authorization.

### Pipeline Stages

1. **Quick rules filter** — Deterministic, no AI call. Catches obvious patterns (known noise signatures, duplicate detection within time window).
2. **Heuristic scoring** — Calculates risk score using event severity, recurrence patterns, geofence proximity, time-of-day factors. No external API call.
3. **AI textual/contextual evaluation** — Sends structured context to the AI SDK agent for classification, confidence scoring, and explanation generation.
4. **Multimodal evaluation** — If media is available, sends images/video/audio to a multimodal model for visual/audio validation.
5. **Decision fusion** — Combines all signals (rules, heuristics, AI text, media) into the final `AIEventEvaluation` record with weighted confidence.

### Performance

- `ai_event_evaluations` indexed on `(team_id, evaluated_at)` for dashboard queries and `(normalized_event_id)` for event lookups.
- AI inference logs grow rapidly — consider archiving logs older than 90 days to cold storage.
- Cache `TenantAIProfile` in Valkey with 5-minute TTL to avoid repeated database reads during batch evaluations.
- AI SDK agent calls have a 120-second timeout; media evaluations have 180 seconds to accommodate large files.

### Idempotency

- Usage events use `event_key` format: `ai_call:{evaluationId}`, `ai_tokens_in:{inferenceLogId}`, `ai_tokens_out:{inferenceLogId}`.
- Reevaluation requests are deduplicated by checking for existing `pending` or `processing` requests for the same `normalized_event_id` and `trigger_type`.
